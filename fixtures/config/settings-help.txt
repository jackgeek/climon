  version
    Type: number; Default: 1; Scope: client, daemon, server; internal
    Schema version for the persisted config file format. Always 1 for the current
    release.

  server.host
    Type: string; Default: 127.0.0.1; Scope: server
    IP address the dashboard server binds to. Defaults to loopback for local-only
    access.

  server.port
    Type: number; Default: 3131; Scope: server
    TCP port the dashboard server listens on. Change if 3131 conflicts with another
    service.

  terminal.clampBrowserToHost
    Type: boolean; Default: false; Scope: daemon
    When false (default), a browser viewer may grow the shared PTY beyond the host
    terminal's dimensions. Set true to clamp viewer size to the host terminal to prevent
    content mangling.

  terminal.detachPrefix
    Type: number; Default: 28; Scope: client
    Byte value of the detach key prefix (default 0x1c = Ctrl-\). Press prefix then 'd'
    to detach without stopping the command. Must be an integer in [0, 255].

  hotKeys.focusTopSession
    Type: string; Default: Alt+J; Scope: server, browser
    Web dashboard shortcut that selects the top session in the list and focuses its
    terminal. Format is "Mod+...+Key" (e.g. "Alt+T", "Ctrl+Shift+J"). Set to an empty
    string to disable.

  dashboard.theme
    Type: string; Default: Default; Scope: server, browser
    Default web dashboard terminal colour theme (by display name, e.g. "Dracula").
    Sessions without their own theme inherit this. Choose from the dashboard "Default
    theme" picker; defaults to "Default".

  dashboard.keyBarPinned
    Type: boolean; Default: true; Scope: server, browser
    Whether the web dashboard key bar is pinned open.

  dashboard.stateIconNoMotion
    Type: boolean; Default: false; Scope: server, browser
    When true, the web dashboard freezes the animated terminal-progress indicator (OSC
    9;4 indeterminate spinner) into a static icon, honouring reduced-motion preferences.
    Defaults to false (animated).

  attention.idleSeconds
    Type: number; Default: 10; Scope: daemon
    Number of seconds the rendered terminal grid must remain unchanged before the
    session is flagged as needing attention. Set to 0 or negative to disable
    static-screen detection.

  notifications.smartSnippet
    Type: boolean; Default: true; Scope: daemon
    When true (default), attention notifications include a fuzzy-extracted snippet of
    the last relevant terminal output as the notification body. Set false to send only
    the session name / terminal title.

  remote.enabled
    Type: boolean; Default: unset; Scope: client
    Enables remote uplink so the local devbox forwards session metadata and I/O to a
    remote dashboard over a dev tunnel or direct connection.

  remote.host
    Type: string; Default: unset; Scope: client
    Direct remote uplink host for same-machine or LAN setups. Takes precedence over dev
    tunnel forwarding when set.

  remote.ingestHost
    Type: string; Default: unset; Scope: client
    Host address where the dashboard-side ingest daemon should listen for incoming
    remote session connections.

  remote.tunnelId
    Type: string; Default: unset; Scope: client
    Dev tunnel id (e.g. "happy-tree-abc123") used by `devtunnel connect` to forward
    local climon traffic to a remote dashboard.

  remote.dashboardTunnelId
    Type: string; Default: unset; Scope: server; internal
    Server-owned persisted dashboard tunnel id used to reuse tunnel identity for tunnel
    link sessions.

  remote.dashboardTunnelCluster
    Type: string; Default: unset; Scope: server; internal
    Server-owned persisted dashboard tunnel cluster used to reuse tunnel identity for
    tunnel link sessions.

  remote.dashboardTunnelEnabled
    Type: boolean; Default: unset; Scope: server; internal
    Server-owned flag recording whether the Tunnel Link is enabled, so the server
    re-establishes the dashboard tunnel automatically on startup.

  remote.port
    Type: number; Default: unset; Scope: client
    Local port the devbox forwards and the ingest daemon listens on. Defaults to
    server.port if not explicitly set.

  remote.ingestPortRetryAttempts
    Type: number; Default: 100; Scope: server
    How many consecutive ports the ingest daemon will try, starting at its preferred
    port, before giving up. Raise it if many ports near the default are already in use.

  remote.clientId
    Type: string; Default: unset; Scope: client
    Stable, non-secret client namespace identifying this machine's sessions. Defaults to
    the machine hostname when unset; set it to a value that is unique per host to avoid
    session ID collisions across machines.

  remote.spawnSecret
    Type: string; Default: unset; Scope: client, server; sensitive
    Shared HMAC secret authenticating dashboard→devbox spawn commands. Generated
    automatically on the dashboard host when feature.remoteSpawn is enabled, and planted
    on the devbox by the remotes-screen setup script. Keep it secret.

  remote.keepAlive
    Type: number; Default: 60; Scope: client
    Interval in seconds between mux keepalive pings sent over the remote uplink/ingest
    connection. Prevents dev tunnel idle timeouts from dropping the connection. Set to 0
    to disable.

  remote.peerHome
    Type: string; Default: unset; Scope: client, server
    Path to the peer OS's CLIMON_HOME for same-machine WSL<->Windows discovery (e.g.
    /mnt/c/Users/<you>/.climon from WSL, or \\wsl.localhost\<distro>\home\<you>\.climon
    from Windows). When feature.wslBridge is enabled, climon reads the peer's beacons
    and wires sessions to it. Usually set automatically by `climon link`.

  remote.peerHost
    Type: string; Default: unset; Scope: client, server
    Optional host override used to reach the peer dashboard/ingest. Leave unset to
    auto-detect (localhost, or the WSL gateway IP under NAT networking).

  remote.autoLink
    Type: boolean; Default: true; Scope: client
    When true (default), the first `climon` run inside WSL attempts to auto-link to a
    Windows-side climon by detecting its CLIMON_HOME and setting remote.peerHome on both
    sides. Auto-link configures discovery only; it never enables feature.wslBridge. Set
    false to disable auto-linking.

  session.color
    Type: string; Default: auto; Scope: client, daemon, server
    Specifies the default accent color for new sessions. Accepts ANSI color names (red,
    green, etc.), 'none', or 'auto' for automatic assignment.

  session.priority
    Type: number; Default: 500; Scope: client, daemon, server
    Default sort priority (0-1000) for new sessions. Lower numbers sort first within
    each status group.

  session.terminalProgram
    Type: string; Default: unset; Scope: client
    Command template used to open a terminal window for a non-headless (visible) session
    spawned from the dashboard. Use the {cmd} placeholder for the climon command to run.
    When unset, climon auto-detects a terminal per OS (Terminal.app, Windows Terminal,
    or x-terminal-emulator/gnome-terminal/konsole/xterm).

  tunnelLink.keepAlive
    Type: number; Default: 60; Scope: server
    Interval in seconds between keep-alive pings sent through the Tunnel Link dev tunnel
    relay to prevent idle disconnection. Set to 0 to disable keep-alive pings.

  logging.level
    Type: string; Default: trace; Scope: client, daemon, server
    Minimum log level emitted by climon processes. One of: trace, debug, info, warn,
    error, fatal, silent. Defaults to trace (everything). Set to silent to disable
    logging. Overridden per-invocation by the CLIMON_LOG_LEVEL environment variable.

  feature.sessionSpawning
    Type: string; Default: disabled; Scope: client, daemon, server, browser
    Allow spawning new sessions from the dashboard. Set to "enabled" or "disabled".
    [status: experimental]

  feature.remoteSpawn
    Type: string; Default: disabled; Scope: client, daemon, server, browser
    Allow the dashboard to spawn sessions on remote devboxes over a signed,
    replay-protected mux command channel. Set to "enabled" or "disabled". [status:
    experimental]

  feature.wslBridge
    Type: string; Default: disabled; Scope: client, daemon, server, browser
    Stream sessions between a same-machine WSL distro and Windows so they appear on one
    shared dashboard. Set to "enabled" or "disabled". [status: untested]

  feature.remotes
    Type: string; Default: disabled; Scope: client, daemon, server, browser
    Connect sessions from a remote devbox to this dashboard over the ingest/uplink
    bridge. Set to "enabled" or "disabled". [status: experimental]

  telemetry.enabled
    Type: boolean; Default: false; Scope: client, server
    When true, climon sends anonymous, opt-in usage telemetry keyed only by a random
    install id (no PII, session output, commands, paths, or hostnames). Off by default.

  update.auto
    Type: boolean; Default: false; Scope: client
    When true, climon downloads and applies signed updates automatically in the
    background. When false (default), it only prints a one-line banner suggesting
    `climon --update`.

  update.lastCheck
    Type: string; Default: unset; Scope: client; internal
    ISO-8601 timestamp of the last background update check. Used to throttle checks.

  update.availableVersion
    Type: string; Default: unset; Scope: client; internal
    Latest version discovered by the background update check, if newer than the
    installed version. Cleared after a successful update.

  license.noticeShown
    Type: boolean; Default: unset; Scope: client; internal
    Whether the one-time MIT license-change notice has been shown. Set automatically the
    first time an install that upgraded from a pre-open-source (EULA-gated) build
    launches; never shown on fresh installs.

  install.id
    Type: string; Default: unset; Scope: client, server; internal
    Anonymous, randomly generated install identifier used only when telemetry is
    enabled. Contains no personal information.
